Notes

  • Select a tagged PDF file to upload. Drag and drop the PDF file, or browse using the Browse button.
  • By default the generated XML will follow any specified Role Mappings so the generated elements should use names from the standard PDF specifications. If the Follow Role Map `option is unchecked, the generated elements will use the names and namespaces specified in any locally defined Structure elements. The LaTeX-Project Structure Elements are declared in the schema used here, but other namespacees result in XML files thatwill not validate to the supplied schema.
  • The default PDF/UA-2 schema uses the PDF 2 and PDF 1 Structure element namespaces. You may instead select a schema for PDF/UA-1 which declares PDF 1.7 Strutcure elements (in no-namespace). You may also select None to suppress schema validation.

Sources

This web version is intended for small illustrative examples only, and is provided as a free service.

We encourage use of a local installation to allow easier experimentation and to reduce load on this server.

The full sources are available under Open Source licences.

  • show_lua_tags Is a Lua script that just requires the texlua executable that is included in all modern TeX distributions. Source and documentation are on GitHub.
  • The Relax NG Schema are currently developed as part of the LaTeX-Project tagging project, also on GitHub
  • Any Relax NG validator can be used (RNV and Emacs/nxml are used in development). This web form uses RNV available from davidashen.net.
